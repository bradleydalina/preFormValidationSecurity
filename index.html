Overview

Dalina Form Validation Security is a lightweight JavaScript class helper designed to validate form inputs before they are submitted to the server. It helps stop common client-side attacks, invalid data, and tampering by enforcing rules directly in the browser.

It enforce validation rules, and block unsafe or suspicious values in real time. By validating and filtering data at the earliest point—the form itself—the library helps improve data quality, reduce server load, and add an extra layer of protection against common client-side attacks.

Dalina Form Validation Security:

What the Library Does
  • Validates required fields, formats, and input patterns
  • Detects invalid, empty, or malformed data
  • Prevents form submission when validation fails
  • Helps block basic injection attempts and input tampering
  • Provides instant feedback to users

It runs automatically when a form is submitted or when inputs change, making it easy to integrate with any website or web application.

Features

Dalina Form Validation Security provides a powerful yet lightweight set of tools to protect and validate your web forms before submission.

  • Real-time input validation Instantly checks user input as it is entered or submitted.
  • Pre-submission security checks Blocks unsafe, empty, or malformed data before it reaches your server.
  • Lightweight and fast Designed to run with minimal impact on page performance.
  • Easy to integrate Works with plain HTML forms without needing frameworks.
  • Custom validation rules Allows you to define your own rules for specific fields.
  • Error handling Highlights invalid fields and prevents form submission.
  • Helps prevent common attacks Adds a first layer of defense against basic injection and input tampering.
  • Improves user experience Gives users immediate feedback, reducing frustration and errors.

Core Concepts

Dalina Form Validation Security is built around one simple idea: validate and secure form data before it ever reaches your server.

Instead of waiting for backend errors, the library checks inputs directly in the browser, stopping bad or unsafe data at the source.

  1. Form as the Security Gate

    Every form becomes a security checkpoint.

    When a user submits a form:

    • All input fields are inspected
    • Values are checked against validation rules
    • Unsafe or invalid data is blocked
    • Submission is allowed only if everything passes

    This prevents bad data from being sent in the first place.

  2. Automatic Field Detection

    The library automatically scans:

    • <input>
    • <textarea>
    • <select>

    It reads attributes like:

    • required
    • type
    • min, max
    • pattern

    This allows it to validate most fields without extra configuration.

  3. Rules-Based Validation

    Each field is validated using simple rules such as:

    • Required
    • Email format
    • Number limits
    • Custom patterns

    You can also define your own rules for special fields.

  4. Real-Time Protection

    Validation happens:

    • When the user types
    • When a field loses focus
    • When the form is submitted

    This gives users instant feedback and prevents accidental mistakes.

  5. First Layer of Security

    Pre Form Validation Security acts as a front-line defense:

    • Stops empty or malformed input
    • Reduces injection attempts
    • Prevents obvious tampering

    It does not replace server-side validation—but it greatly reduces bad data before it reaches your backend.

Configuration

Dalina Form Validation Security can be customized by passing a configuration object when you initialize the class. This allows you to control how validation and security checks behave for each form.


					
1new DalinaFVS('#loginForm', {

					
2	bait:false,

					
3	defaultKey:000-00-0000

					
4	log:true,

					
5	debug:true

					
6	style:`input:user-invalid{border:solid 1px #ff0000;}`

					
7})
Option Type Description
bait Boolean This is adds two hidden inputs fields, one that is not required input and must not have a value when submitted, the other one must have a default value and must not be modified, once modified it will indicate as bot activity.
defaultKey String/Number A default value that helps detect a boot requests.
log Boolean Once set to true, it will use the console to log every validation within the form using this helper.
debug Boolean Once set to true, it will use the console to log every error catch during runtime.
style string It allows you to add custom css styles for handling validtion indicator.

Events & Hooks

Dalina Form Validation Security provides hooks (callbacks) that let you run your own code when important things happen — such as validation, errors, or successful form submission.

These hooks allow you to connect your UI, logging, or custom security logic to the library.

How Hooks Work

You can call the hooks by chaining it:


								
1new DalinaFVS('#loginForm')

                
2  .addValidator( function(){ ... } )

								
3  .onValidate( function(){ ... } )

								
4  .onSuccess( function(){ ... } )

								
5  .onError( function(){ ... } )

                
6})

Each hook runs automatically at the correct moment.

Available Hooks

Callback/Hook Description
init Automatically fires when you invoke the main function
addValidator Where you can add a custom and complex input validation
onValidate Fires when you click submit button, where all inputs are validated
onSuccess Fires after a successful validation
onError Fires after failed validation
destroy Free memory the library
reset Resets the form and its validation indicators
clear Clear the validation indicators only
refresh Resets the library and reinitialize (.destroy().init())
getData Return form input values in object form {}
getErrors Returns error object messages {}

Validation Flow

  • Initialize DalinaFVS

    Set configs, Check/Verify Form Existence, Add Input Listeners, Add Submit Handlers

  • User Submits Form

    [ button click(), Call .onValidateCallback() ]

  • DalinaFVS Intercepts Submit

    [ activate the input listeners, and submit handlers ]

  • Run Bowser Default Validation and Custom Validators

    [ check Validity, report validity ]

  • Display Indicators

    [ logs data or errors options.log && options.debug ]

  • Is Form Valid?

    • Validation Passed

      [ call onSuccessCallback ]

    • Validation Failed

      [ call onErrorCallback ]

    • Allow Form Submit

      [ if onSuccessCallback not set ]

    • Prevent Form Submit Failed

      Return/Log Errors

Error Handling

This library uses a safe and predictable error-handling system to protect both developers and users.

When debug mode is enabled, configuration problems such as invalid form selectors, wrong hook types, or incorrect validator definitions will throw clear JavaScript errors in the console. This helps developers quickly find and fix issues during development.


						
1const loginFVS = new DalinaFVS('#loginForm', {debug:true});

When debug mode is disabled (production mode), this library does not break the page, allowing the application to respond without exposing technical details to users.

All validation failures are handled through controlled result states rather than unhandled exceptions. This ensures that the form remains secure, stable, and predictable in every situation.


						
1loginFVS.onError( function(errors){ console.log(errors) } )

						

											
1loginFVS.onError( ()=> console.log(loginFVS.getErrors()); )

											

												
1console.log(loginFVS.getErrors());

In short, DalinaFVS provides strict errors for developers and safe behavior for users, making it suitable for both testing and live production environments.
